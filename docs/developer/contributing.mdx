## Before You Start

1. Check [existing issues](https://github.com/pinkfuwa/llumen/issues)
2. Read the code of conduct
3. Fork the repository
4. Set up your [development environment](/developer/building)

## Code Guidelines

### Rust

- No `unwrap()` or `panic!()` in production code
- Use `?` for error propagation
- Document public APIs
- Write tests for new features
- Follow project style (`cargo fmt`)

### JavaScript/Svelte

- Handle errors properly
- Use async/await over promises
- Minimize dependencies
- Keep components small and focused

## Pull Request Process

1. Create branch: `git checkout -b feature/your-feature`
2. Make changes with tests
3. Test: `cargo test && npm test`
4. Commit: `git commit -m "feat: add feature"`
5. Push and create PR
6. Address review feedback
